# Produto e MVP

Esta pasta concentra o que ajuda o projeto novo a nascer com direção clara.

Aqui entra:
- visão do produto
- recorte do MVP
- jornada principal
- blocos de valor
- base de leitura para humano e para o Codex

Ordem de leitura recomendada:
1. `visao_do_produto.md`
2. `mvp_e_fases.md`
3. `jornada_principal.md`
4. `o_que_manter_do_legado.md`

Regra desta pasta:
- linguagem de produto
- pouco jargão técnico
- foco no que o sistema precisa entregar
- nada aqui deve prender a arquitetura nova ao Apps Script
