# Backlog real

Esta pasta organiza a leitura do backlog real do projeto tomando o GitHub como fonte principal.

## Regra

- issue aberta nao significa implementado
- PR merged nao significa backlog refletido
- so ha `completo` quando existe evidencia objetiva no codigo integrado
