# Não faça isso

- não ligar UI direto no Apps Script
- não criar fallback em planilha na nova versão
- não espalhar lógica de ambiente por tela
- não acoplar componente visual a fetch direto
- não criar abstração genérica demais cedo
- não fazer importação real antes do preview mockado existir
- não tentar resolver IA, benchmark, importação e histórico tudo ao mesmo tempo
- não subir PRD sem validar a UI inteira com mock realista
- não deixar o Codex decidir regra de negócio crítica sozinho
- não confundir HML com PRD
